# asset

画像の最適化およびディレクトリ同期管理システム。

## ディレクトリ構成

```text
asset/
├─ css
├─ :
├─ images/ (gitignore)      # 実際に表示で使用されるWebP画像
│ ├─ xxx.webp
│ └─ yyy.webp
├─ images.bak/              # 管理用の元画像
│ ├─ xxx.big.png
│ └─ yyy.jpg
├─ optimize.ipynb           # 最適化スクリプト
├─ .gitignore
└─ README.md
```

---

## 運用ワークフロー

### 1. 初回利用時（同期）

クローンやフェッチの直後は、`.gitignore` の対象である `images/` ディレクトリが存在しない状態です。
全ての画像の参照が切れているため、各自 **`optimize.ipynb`** を実行して表示用の画像を生成してください。

### 2. 画像を追加・変更する場合

1. **`images.bak/`** に元画像を追加してください。
2. スクリプトを実行すると、`images/` に最適化された WebP 画像が生成されます。

> **推奨形式** : `png` または `svg`
>
> `jpg` は最適化時に再圧縮による画質劣化が重なるため、推奨されません。
> また、現状のスクリプトは `webp` ファイルを処理対象外としているため、手動で `webp` を入れる場合は最適化済みであることを確認してください。

## 画像のリサイズ指定（サイズ指定子）

ファイル名の拡張子の直前に特定の文字列を付与することで、最適化時の最大サイズを制限できます。
例: `sample.med.png` → `sample.webp` (最大長辺 800px)

| 指定子     | 内容 | 最大サイズ (長辺) | 用途イメージ        |
|:--------|:---|:-----------|:--------------|
| **sml** | 大  | 640px      | アイコン、アバターなど   |
| **med** | 中  | 1280px     | 記事内画像、一般的な表示  |
| **big** | 小  | 1920px     | フル画面、メインビジュアル |
| (なし)    | -  | 制限なし       | オリジナルサイズを維持   |

* ※指定子は最適化処理時に自動的に削除されます。
* ※サイト上での表示サイズを想定し、適切な指定子を付与してください。

## スクリプトの実行方法 (VSCode)

### 事前準備

1. **Pythonのインストール**: PCにインストールしてください。環境変数（Path）への追加を忘れないよう注意してください。
2. **VSCode拡張機能の導入**: 以下の2つをインストールしてください。
    - `Python` (Microsoft)
    - `Jupyter` (Microsoft)

### 実行手順

1. VSCodeで `optimize.ipynb` を開きます。
2. 画面上部の **「▷ Run All（すべて実行）」** をクリックします。
3. Pythonカーネルの選択を求められた場合は、インストールしたPython環境を選択してください。
4. 処理が完了するまで待機します（反応がない場合は、再度「Run All」を押してください）。
