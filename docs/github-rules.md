## git使いかた

### github flow上の運用
-1.目的と仕様をそろえる
タスクを小さく分けて、何を作るか・完了条件をチームで一致させます。

-2.作業チケットを作る
Issueやタスク管理ツールで、担当・期限・優先度を明確にします。）

### ここからgitの上の運用

-3.ブランチを分けて実装する
mainで直接作業せず、機能ごとに作業ブランチで進めます。

<ブランチ命名規則>(英語推奨)
機能作成：feature/機能名
バグ修正：bugfix/内容
リファクタリング：refactor/内容

-4.こまめにコミットする
変更内容が追える単位でコミットし、メッセージは何をしたか分かる形にします。

<コミットメッセージ例>(接頭語：英語、内容：日本語推奨)
feat: 新しい機能
fix: バグの修正
docs: ドキュメントのみの変更
style: 空白、フォーマット、セミコロン追加など
refactor: 仕様に影響がないコード改善(リファクタ)
perf: パフォーマンス向上関連
test: テスト関連
chore: ビルド、補助ツール、ライブラリ関連 

-5.リモートへpushしてPRを作る
作業ブランチをpushし、Pull Requestでレビュー依頼します。

＜プルリクのコメント例＞
- 何をしたか
- 変更の理由や背景
- 注意点や確認してほしい箇所

6.コードレビューを行う
PRに対して、レビュアーがコードを確認し、
バグ、設計、可読性、命名、テスト不足を確認して指摘・修正します。

7.mainへマージ
レビュー承認後にマージし、不要ブランチを削除します。
そして、更新されたmainをpullしてローカルも最新にします。

### git の基本コマンド
クローン
- git clone <リポジトリURL>：リポジトリをローカルにコピー

ブランチ関連
- git status：現在のブランチと変更状況を確認
- git branch：ローカルのブランチ一覧を表示
- git switch -b <ブランチ名>：新しいブランチを作成して切り替え
- git switch <ブランチ名>：既存のブランチに切り替え

コミット関連
- git add <ファイル名>：変更をステージングエリアに追加
- git commit -m "コミットメッセージ"：変更をコミット
- git push origin <ブランチ名>：リモートにブランチをアップロード
- git pull origin main：リモートのmainブランチをローカルに取り込む

マージ関連
- git merge <ブランチ名>：指定したブランチを現在のブランチにマージ
- git branch -d <ブランチ名>：ローカルのブランチを削除
- git push origin --delete <ブランチ名>：リモートのブランチを削除
